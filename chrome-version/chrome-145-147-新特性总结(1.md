# Chrome 145–147 完整新特性总结

> 整理自 Chrome for Developers 官方博客及完整 Release Notes，涵盖 Chrome 145（2026-02-10）、146（2026-03-10）、147（2026-04-07）三个版本的所有重要更新。

---

## Chrome 145（2026 年 2 月）

### CSS & UI

#### 1. CSS 多列布局换行（`column-wrap` / `column-height`）

新增 `column-wrap` 和 `column-height` 属性（来自 CSS Multicol Level 2 规范），支持竖向的多列换行布局。此前溢出内容只能横向排列产生水平滚动条，现在可以自动折到下一行显示。

```css
.container {
  columns: 3;
  column-height: 200px;
  column-wrap: wrap;
}
```

#### 2. `text-justify` 属性支持

可以通过 `text-justify` 控制 `text-align: justify` 的对齐方式，例如强制对英文文本也展开字符间距进行两端对齐。

#### 3. `letter-spacing` / `word-spacing` 支持百分比值

`letter-spacing` 和 `word-spacing` 现在可以使用百分比值，相对于空格字符（U+0020）的宽度计算。对响应式排版更友好，字距可随字体大小自动伸缩。

```css
p {
  letter-spacing: 5%;
  word-spacing: 10%;
}
```

#### 4. 高 `border-radius` 时阴影边缘精确计算

当 `border-radius` 接近 50%（近圆形）时，阴影和裁切边界现在能精确贴合曲线轮廓，消除视觉偏差。也同样适用于 `corner-shape` 定义的非圆曲线轮廓。

#### 5. 可定制 `<select>` 列表框（Customizable Select Listbox）

`appearance: base-select` 现在也支持 `<select multiple>` 和 `<select size=N>` 的列表框渲染模式，允许对这两种多选样式进行深度定制。

#### 6. `focus()` 方法新增 `focusVisible` 选项

调用 `element.focus({ focusVisible: true })` 时强制显示焦点环，`focusVisible: false` 时强制隐藏，与 `:focus-visible` 伪类联动。

#### 7. 强制颜色模式下 Emoji 单色渲染

在强制高对比度（Forced Colors）模式下，`font-variant-emoji` 计算值为 `normal` 或 `unicode` 的 Emoji 会自动切换到单色字形，确保符合系统高对比度颜色要求。

#### 8. 非根滚动容器的弹性过滚动效果（Overscroll）

嵌套滚动元素到达边界时，弹性过滚动视觉效果现在直接作用于该元素自身，而不再"穿透"到根滚动容器。可通过 `overscroll-behavior` 逐元素控制。

---

### JavaScript

#### 9. Map/WeakMap Upsert（`getOrInsert` / `getOrInsertComputed`）

ECMAScript 新提案，新增以下方法：
- `Map.prototype.getOrInsert(key, defaultValue)`
- `Map.prototype.getOrInsertComputed(key, fn)`
- `WeakMap.prototype.getOrInsert` / `getOrInsertComputed`

查找 key 时若不存在则自动插入默认值，简化"读-插"操作。

```js
const map = new Map();
map.getOrInsert('count', 0);
map.getOrInsertComputed('list', () => []);
```

#### 10. Crash Reporting Key-Value API（`window.crashReport`）

新增 `window.crashReport` 键值 API，开发者可向其写入应用状态信息。当渲染进程崩溃时，这些数据会附加到 `CrashReportBody` 中，便于调试崩溃原因。

#### 11. User-Agent 字符串默认精简（移除 `UserAgentReduction` 策略）

从 Chrome 145 起，`UserAgentReduction` 企业策略被移除，Chrome 默认发送精简版 UA 字符串。建议迁移到 **User-Agent Client Hints（UA-CH）** 来按需获取浏览器和设备信息。

#### 12. Navigation API：`navigation.transition` 中暴露目标（`to`）

`NavigationTransition` 新增 `to` 属性（`NavigationDestination` 类型），在预提交（precommit）阶段即可获知导航目标 URL，无需等待页面切换完成。

#### 13. Cookie Store API 支持 `maxAge` 属性

使用 Cookie Store API 设置 Cookie 时，现在可以用 `maxAge`（秒数）指定有效期，与 `document.cookie` 和 `Set-Cookie` HTTP 头的 `Max-Age` 行为对齐。

#### 14. 安全支付确认（SPC）：设备绑定密钥 + UX 刷新

- 新增浏览器绑定密钥（Browser Bound Keys），为 SPC 断言加入额外的设备级加密签名，私钥不跨设备同步。
- Android Chrome 上的 SPC 对话框 UX 全面更新，支持商户徽标展示、更丰富的支付信息展示，以及更细粒度的用户取消/继续状态区分。

---

### 安全

#### 15. Origin API

引入原生 `Origin` 对象，封装 Web 源（origin）概念，提供比较、序列化、解析等方法，解决开发者长期以来只能操作字符串 origin 导致同源比较出错的问题。

#### 16. 设备绑定会话凭证（DBSC）

将用户 Session 绑定到特定设备的硬件密钥，通过定期验证私钥来刷新短周期 Cookie，从根本上防止 Cookie 窃取后在其他设备使用。

#### 17. 本地网络访问权限拆分（LNA Split Permissions）

将原来统一的 `local-network-access` 权限拆分为两个更细粒度的权限：
- `local-network`：访问本地地址空间 IP
- `loopback-network`：访问回环地址空间（loopback）

---

### 性能

#### 18. 导航性能置信度字段（`confidence`）

`PerformanceNavigationTiming` 新增 `confidence` 字段，帮助开发者判断当前采集到的导航性能数据是否有代表性（区分冷启动与热启动等双峰分布场景）。

#### 19. 性能条目新增 `paintTime` 和 `presentationTime`

在 Element Timing、LCP、Long Animation Frames、Paint Timing 中新增：
- `paintTime`：渲染阶段结束、浏览器开始绘制的时间点
- `presentationTime`：像素实际到达屏幕的时间点

#### 20. LayoutShift API 改用 CSS 像素

Layout Instability API 中的位移归因数据（`prevRect` / `currentRect`）改为以 CSS 像素上报，与其他布局相关 API 保持一致，简化调试。

---

### 存储

#### 21. IndexedDB：内存上下文切换到 SQLite 后端

隐身模式等内存上下文中，IndexedDB 的底层实现已从 LevelDB + 扁平文件迁移到 SQLite，提升可靠性与性能。Web API 无变化。

---

### 多媒体

#### 22. WebRTC `VideoFrame.metadata()` 暴露 `rtpTimestamp`

来自 WebRTC 源的视频帧现在可通过 `VideoFrame.metadata()` 获取 `rtpTimestamp` 字段，便于音视频同步和时间戳分析。

---

### Origin Trial（实验特性）

#### 23. JPEG XL 图像解码支持（`image/jxl`）

通过基于 Rust 的纯安全解码器 `jxl-rs` 支持 JPEG XL 格式，提供渐进解码、宽色域/HDR/高位深、动画等特性。需开启 flag 使用。

#### 24. WebAudio 可配置渲染量子（Render Quantum）

`AudioContext` 和 `OfflineAudioContext` 新增 `renderSizeHint` 选项，允许指定渲染帧数（`default` 为 128 帧，或传入 `"hardware"` 让浏览器自动选择）。

---

## Chrome 146（2026 年 3 月）

### CSS & UI

#### 1. 滚动触发动画（Scroll-triggered Animations）

通过 CSS 声明式语法，在滚动到指定位置时触发动画的播放、暂停或重置，无需 JavaScript 监听滚动事件。浏览器可将此类交互卸载到 Worker 线程，性能更好。也提供了 JS 接口以扩展到 Web Animations。

```css
@keyframes slide-in {
  from { transform: translateX(-100%); }
  to { transform: translateX(0); }
}
.box {
  animation: slide-in linear;
  animation-trigger: scroll() entry;
}
```

#### 2. `trigger-scope` 属性

配合滚动触发动画，`trigger-scope` 可将动画触发器名称的可见范围限制在特定作用域内，避免全局名称污染，与 `anchor-name` 作用域隔离机制类似。

#### 3. `<meta name="text-scale">`

允许页面声明自身支持用户文字大小偏好缩放（通过 `rem`/`em` 布局），浏览器会相应地缩放根元素默认字号，并禁用全页缩放和移动端文字自动调整等旧机制。

#### 4. 作用域自定义元素注册表（Scoped Custom Element Registry）

允许在同一页面为同一标签名注册多个不同的自定义元素定义，解决多组件库的命名冲突问题。

```js
const registry = new CustomElementRegistry();
registry.define('my-btn', MyButton);
// 仅在绑定的 shadow root 内生效
```

---

### JavaScript

#### 5. Iterator 序列化（`Iterator.concat`）

TC39 新提案，`Iterator.concat(...iterables)` 将多个迭代器按顺序连接成一个新迭代器，无需手动实现拼接逻辑。

```js
const combined = Iterator.concat([1, 2], [3, 4], [5]);
[...combined]; // [1, 2, 3, 4, 5]
```

#### 6. 选择性权限拦截（Selective Permissions Intervention）

当页面授予了某个强大 API 权限（如蓝牙、摄像头、麦克风、地理位置等）时，页面内运行的广告脚本将无法滥用这些权限，保护用户隐私不被第三方脚本偷用。

#### 7. 拖放 API `dropEffect` 值保留修复

`dragover` 事件中设置的 `dataTransfer.dropEffect` 值，现在能正确保留并在后续 `drop` 事件中读取，修复了 Chromium 长期不符合 HTML5 规范的问题。

---

### 多媒体

#### 8. WebAudio 播放统计 API（`AudioContext.playbackStats`）

新增 `AudioContext.playbackStats` 属性，返回 `AudioPlaybackStats` 对象，包含平均延迟、最小/最大延迟、欠载时长、欠载次数等音频质量统计数据，便于监控播放卡顿。

---

### 安全

#### 9. Sanitizer API

浏览器原生 HTML 净化接口，自动移除用户输入 HTML 中可能执行脚本的危险内容，更轻松地构建防 XSS 应用。Firefox 也已同步支持。

```js
const sanitizer = new Sanitizer();
element.setHTML(untrustedHTML, { sanitizer });
```

---

### 网络

#### 10. Data URL MIME 类型参数保留

Data URL 的 `Content-Type` 头现在会保留 MIME 类型参数（如 `charset`、`boundary`），符合 Fetch Standard 规范。

---

### 性能

#### 11. LCP 候选项发射行为修正

LCP 算法改为基于已绘制的最大图像（而非最大待处理图像）发射候选项，与 Firefox/Safari 对齐，减少引擎间差异，提升 LCP 分析准确性。

---

### 图形（WebGPU）

#### 12. WebGPU Texture and Sampler Lets

WGSL 新增 `texture_and_sampler_let` 语言特性，允许将 `texture` 和 `sampler` 对象存入 `let` 声明，提升着色器代码灵活性。

#### 13. WebGPU Transient Attachments

新增 `TRANSIENT_ATTACHMENT` GPUTexture 用途标志，允许渲染通道的附件数据保留在 tile memory 中，无需写回 VRAM，可显著降低带宽消耗，在移动端 GPU 上效果尤为明显。

#### 14. WebGPU 兼容模式（Compatibility Mode）

新增 opt-in 的 WebGPU 兼容子集，支持在 OpenGL ES / Direct3D 11 等旧图形 API 上运行 WebGPU 应用，大幅扩展设备兼容范围。只需在 `requestAdapter` 时指定 `featureLevel: 'compatibility'`。

---

### PWA / Capabilities

#### 15. 文件处理时填充 `LaunchParams.targetURL`

PWA 通过文件处理启动时，`LaunchParams.targetURL` 现在会正确填充 manifest `action` 字段的 URL（此前为 `null`）。

#### 16. 修复页面刷新时重复触发 LaunchParams

用户刷新 PWA 页面时，不再重复向 `launchQueue` 发送上次启动的文件参数，避免文件被重复处理。

---

### Origin Trial（实验特性）

#### 17. WebNN（Web Neural Network API）

允许 Web 应用调用操作系统原生 ML 服务和底层硬件能力（CPU/GPU/NPU），在浏览器内实现高效、可靠的机器学习推理。

#### 18. CPU Performance API

暴露设备 CPU 算力信息，可与 Compute Pressure API 结合使用，让应用根据设备性能动态调整计算密度。

#### 19. 预测规则（Speculation Rules）`form_submission` 字段

允许在 `prerender` 的 speculation rules 中声明 `form_submission` 字段，预加载表单提交后的目标页面（如搜索结果页）。

#### 20. Focusgroup

通过 `focusgroup` HTML 属性，让一组可聚焦元素支持键盘方向键导航，无需手动编写键盘事件处理逻辑。

---

## Chrome 147（2026 年 4 月）

### CSS & UI

#### 1. 元素级视图过渡（Element-scoped View Transitions）

`element.startViewTransition()` 现可在任意 HTML 元素上调用，支持：
- 多个元素的过渡动画**并发执行**
- **嵌套**视图过渡
- 过渡期间页面其余部分保持交互

```js
document.querySelector('.card').startViewTransition(() => updateCard());
```

#### 2. CSS `contrast-color()` 函数

根据给定背景色自动返回对比度最高的黑色或白色，适用于任何颜色值语境，满足无障碍对比度要求。

```css
.button {
  background: var(--brand);
  color: contrast-color(var(--brand)); /* 自动返回黑或白 */
}
```

#### 3. Timeline Named Range `scroll`

为视图时间线（View Timeline）新增 `scroll` 命名范围，与已有的 `entry`、`exit`、`cover`、`contain` 一起，提供更细粒度的滚动动画控制点。

#### 4. CSS `border-shape` 属性

为边框定义任意形状（多边形、圆形、`shape()` 路径等），只影响边框形状和内部裁切，与 `clip-path` 裁切整个元素的行为有本质区别。

```css
.card {
  border: 4px solid blue;
  border-shape: polygon(0 0, 100% 0, 90% 100%, 10% 100%);
}
```

#### 5. `CSSPseudoElement` 接口

新增 `Element.pseudo(type)` 方法，返回代表伪元素的 `CSSPseudoElement` 对象，提供 `type`、`element`、`parent`、`pseudo()` 等属性和方法，支持 `::before`、`::after`、`::marker`。

#### 6. 事件的 `pseudoTarget` 属性

`UIEvent`、`AnimationEvent`、`TransitionEvent` 新增 `.pseudoTarget` 属性，精确指出事件是否发生在伪元素上（如 `::after` 被点击），而不只是其宿主元素。

#### 7. `border-width` / `outline-width` / `column-rule-width` 解耦

这三个宽度属性的计算值不再因对应 `*-style` 设为 `none` / `hidden` 而被强制归零，始终反映作者指定的值。与 Firefox 和 WebKit 行为对齐。

#### 8. SVG `<textPath>` 支持内联 `path` 属性

`<textPath>` 元素现在支持直接通过 `path` 属性内联定义路径几何，无需单独声明 `<path>` 元素并用 `href` 引用。`path` 属性优先级高于 `href`。

---

### JavaScript

#### 9. `Math.sumPrecise`

TC39 新提案，`Math.sumPrecise(iterable)` 对可迭代对象中的数值求和，使用精度更高的算法，避免原生 `+` 累加的浮点误差。

```js
Math.sumPrecise([0.1, 0.2, 0.3]); // 精确返回 0.6
```

---

### 网络 / 本地网络访问（LNA）

#### 10. `Request.isReloadNavigation` 属性

Fetch API `Request` 新增只读布尔属性 `isReloadNavigation`，在 Service Worker 的 `FetchEvent` 中可判断当前请求是否由用户触发的页面刷新引起。

#### 11. LNA 限制扩展至 WebTransport

通过 WebTransport 访问用户本地网络（本地 IP 或回环地址）现在需要权限提示，与 Fetch 请求的 LNA 限制保持一致。

#### 12. LNA 限制扩展至 WebSockets

WebSocket 连接到本地地址现在也会触发权限提示，现有 LNA 企业策略同样适用。

#### 13. Service Worker `WindowClient.navigate()` 的 LNA 限制

Service Worker 通过 `WindowClient.navigate()` 发起的子框架导航现在也受本地网络访问限制约束。

---

### DOM

#### 14. `<link rel="modulepreload">` 支持 JSON 和 CSS 模块

```html
<link rel="modulepreload" as="style" href="styles.css">
<link rel="modulepreload" as="json" href="data.json">
```

补全了 `modulepreload` 对 JSON 和 CSS 模块类型的预加载支持，之前只能预加载 JS 模块。

#### 15. Rust XML 解析器（非 XSLT 场景）

非 XSLT 场景下的 XML 解析已切换到基于 Rust 的安全实现，替换原来 C++ 的 `libxml2`，从根本上消除 XML 解析中的内存安全问题。

---

### 性能

#### 16. Device Memory API 数值更新

设备内存 API 报告的可用值更新为：
- **Android**：1、2、4、8 GB
- **其他平台**：2、4、8、16、32 GB

替换了自 2016 年沿用至今的旧值（0.25–8 GB），降低低端设备的指纹风险，同时更好地区分高端设备。

---

### 设备 / XR

#### 17. WebXR 平面检测（Plane Detection）

获取环境中检测到的平面集合（如墙面、桌面），支持语义标签，比深度感应 API 更完整，即便有物体遮挡也能还原平面的完整边界。

#### 18. WebXR Layers

为沉浸式内容提供更高效的渲染方式，支持系统合成器管理不同类型的层（Layer），原生颜色/深度纹理和纹理数组等，减少 JavaScript 渲染开销。

---

### Origin Trial（实验特性）

#### 19. 预渲染跨域 iframe（Prerender Cross-origin Iframes）

顶层页面 HTTP 响应中添加 `Supports-Loading-Mode: prerender-cross-origin-frames` 头，即可让浏览器预渲染所有跨域 iframe，提升页面加载感知速度。

#### 20. Autofill 事件

新增 `autofill` 事件，开发者可监听浏览器自动填充动作，并在填充完成后通知浏览器，便于动态表单与自动填充协同工作。

#### 21. WebNN（续）

继 Chrome 146 之后继续在 Origin Trial 中测试 Web Neural Network API。

---

### 废弃与移除

**XSLT 中 SVG 内联生成移除：** 移除通过 XSL 样式表将 XML 转换为 SVG 的特殊用法，该特性使用率极低。这是 Chrome 逐步弃用 XSLT 计划的一部分，同步配合 Rust XML 解析器的推出。
